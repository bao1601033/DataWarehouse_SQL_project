# Data Catalog – Gold Layer

## Overview
The **Gold Layer** represents the **business-ready data model** of the data warehouse.  
It is designed to support **analytics, reporting, and decision-making** by providing clean, consistent, and well-structured datasets.

This layer follows a **star schema design**, consisting of:
- **Dimension tables** that describe business entities
- **Fact tables** that store measurable business events

---

## 1. gold.dim_customers

### Purpose
The `gold.dim_customers` table stores **customer master data** enriched with demographic and geographic attributes.  
It serves as a central reference for customer-related analysis across all business domains.

### Columns

| Column Name       | Data Type      | Description |
|-------------------|---------------|-------------|
| customer_key      | INT           | Surrogate key generated by the data warehouse to uniquely identify each customer record. Used for joins with fact tables. |
| customer_id       | INT           | Original numerical identifier assigned to the customer in the source system. |
| customer_number   | NVARCHAR(50)  | Alphanumeric customer reference used for business tracking and integration. |
| first_name        | NVARCHAR(50)  | Customer’s given name as recorded in the source data. |
| last_name         | NVARCHAR(50)  | Customer’s family or surname. |
| country           | NVARCHAR(50)  | Country where the customer resides (e.g., Australia). |
| marital_status    | NVARCHAR(50)  | Marital status of the customer (e.g., Married, Single). |
| gender            | NVARCHAR(50)  | Gender of the customer (e.g., Male, Female, N/A). |
| birthdate         | DATE          | Customer’s date of birth in YYYY-MM-DD format. |
| create_date       | DATE          | Date when the customer record was created in the system. |

## 2. gold.dim_products

### Purpose
The `gold.dim_products` table contains **product master data**, including classification, cost, and lifecycle information.  
It enables product-level analysis such as sales performance by category, product line, or maintenance requirements.

### Columns

| Column Name            | Data Type      | Description |
|------------------------|---------------|-------------|
| product_key            | INT           | Surrogate key uniquely identifying each product record. |
| product_id             | INT           | Original product identifier from the source system. |
| product_number         | NVARCHAR(50)  | Structured alphanumeric product code used for inventory and categorization. |
| product_name           | NVARCHAR(50)  | Descriptive name of the product. |
| category_id            | NVARCHAR(50)  | Identifier representing the product’s category. |
| category               | NVARCHAR(50)  | High-level product classification (e.g., Bikes, Components). |
| subcategory            | NVARCHAR(50)  | Detailed classification within the product category. |
| maintenance_required   | NVARCHAR(50)  | Indicates whether the product requires maintenance (Yes or No). |
| cost                   | INT           | Base or standard cost of the product in whole currency units. |
| product_line           | NVARCHAR(50)  | Product line or series (e.g., Road, Mountain). |
| start_date             | DATE          | Date when the product became available or effective. |

---

## 3. gold.fact_sales

### Purpose
The `gold.fact_sales` table stores **transactional sales data** at the line-item level.  
It captures measurable business events and connects to dimension tables via surrogate keys.

### Columns

| Column Name     | Data Type      | Description |
|-----------------|---------------|-------------|
| order_number    | NVARCHAR(50)  | Unique alphanumeric identifier for each sales order. |
| product_key     | INT           | Foreign key linking to `gold.dim_products`. |
| customer_key    | INT           | Foreign key linking to `gold.dim_customers`. |
| order_date      | DATE          | Date when the order was placed. |
| shipping_date   | DATE          | Date when the order was shipped. |
| due_date        | DATE          | Date when payment for the order is due. |
| sales_amount    | INT           | Total monetary value of the sales line item. |
| quantity        | INT           | Number of units sold. |
| price           | INT           | Price per unit at the time of sale. |

---

## Summary
The Gold Layer provides a **clean, stable, and analytics-ready data model**, ensuring:
- Consistent use of surrogate keys
- Clear separation of dimensions and facts
- Reliable support for BI tools, dashboards, and advanced analytics
